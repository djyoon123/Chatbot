# PRD — 인사관리 FAQ 챗봇

## 배경 / 문제

임직원이 근태·공수·복지 등 인사 관련 질문을 할 때마다 담당자가 규정 문서를 직접 찾아 회신하는
수작업이 반복된다. 규정 문서가 흩어져 있어 누락·오답 위험이 있고, 대응 시간도 길다.

## 목표

- 인사규정 질문을 입력하면 **근거 조항과 출처를 포함한 답변**을 즉시 제공
- 답변의 **인용 정확성을 자동 검증**해 사람의 최종 확인 부담을 최소화
- 규정 문서 수정 시 코드 변경 없이 **자동 반영**
- 사내 인사규정과 **근로기준법(상위 법령)**을 함께 근거로 답하고, 충돌 시 법령 우선 명시

## 비목표

- 실시간 법령 조회 또는 외부 데이터베이스 연동
- 재무·전결규정 등 인사관리 외 영역 (근태·공수·복지로 범위 한정)
- 인사평가·징계 등 민감 의사결정의 자동 처리
- 웹 UI / 모바일 앱 배포 (CLI 데모 범위)

## 사용자 / 이해관계자

- **1차 사용자**: 근태·공수·복지 등 인사 관련 질의를 하는 일반 임직원
- **2차 사용자**: 인사 응대를 담당하는 인사팀 담당자
- **검토자**: 챗봇 답변의 최종 신뢰 여부를 판단하는 담당자 (HITL)

## 핵심 시나리오

1. 임직원이 `"연장근로 수당은 어떻게 계산돼?"`를 입력
2. 시스템이 규정에서 관련 조항을 검색 → 답변 생성 → 인용 검증
3. 임직원이 답변 + 검증 리포트를 확인 후 담당 부서에 전달 또는 자체 해결

## 성공 지표 (KPI)

| 지표 | 목표 |
|------|------|
| 규정 밖 내용 생성 | 0건 (또는 `(확인 필요)` 표기) |
| 출처 표기 정확도 | 100% |
| 검증 리포트 제공률 | 100% |
| 규정 수정 반영 시간 | 즉시 (재시작 불필요) |

## 제약 / 리스크

| 리스크 | 완화 방안 |
|--------|-----------|
| 규정 밖 환각 | CoT 자기점검 + 검증 에이전트 |
| 검색 누락 | 바이그램+토큰 복합 점수, k=4 |
| 규정 개정 미반영 | 마크다운 문서 수정 시 자동 반영 |
| 사내규정이 근로기준법 위반 | 근로기준법 통합 인덱싱 + 법령 우선 원칙 명시 |
| 법령 원문 갱신 | `build_law.py` 재실행으로 최신 조문 반영 |
| API 비용 | 시스템 프롬프트 캐싱 (ephemeral) |
